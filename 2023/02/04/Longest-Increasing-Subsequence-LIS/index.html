Longest Increasing Subsequence (LIS)

Question: Given a sequence, fin the length of the longest subsequence that all elements in the subsequence is increasing.

For example, given a sequence as follow: \[ 4 {\quad} 10 {\quad} 15 {\quad} 7 {\quad} 5 {\quad} 3 {\quad} 2 {\quad} 8 {\quad} 9 {\quad} 8 {\quad} 15 {\quad} 17 \] The answer of this sequence is 6. This subsequence is: 4 7 8 9 15 17.

Solution:

Let's think about the brute-force solution. For each elements we have two choices: use it or not. Thus, if we use the brute-force solution the time complexity at least O(2n) .

Commonly speaking, the size of n will larger than 1000. It is a tremendous amount of calculation for computer. Therefore, we have to find out better way to solve it.

We are going to use Dynamic Programming (DP) to solve this problem.

The first step is to define this question by a function. Let's set f(x) means all collection of subsequence ending with the xth element.

The second step is to find out a meaning for the function. Each value in the function has its own meaning. In this question we are finding the longest length of subsequence; thus, we can set the meaning for each value of f(x) to be maximum. f(x) represents the longest length of subsequence which is ending with the xth element.

Next step is to find out how we can solve this function. This is the most important part in DP. Let's assume us have the value of f(0) to f(i-1). How can we get the value of f(i)? Every subsequence ending with the element that is before ith element and smaller than ith element can add the ith element into the end of subsequence.
